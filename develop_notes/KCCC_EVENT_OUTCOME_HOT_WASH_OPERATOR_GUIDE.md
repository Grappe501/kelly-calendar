# Event Outcome / Hot Wash — Operator Guide

## When to use

After an Event’s scheduled end (or when cancelled/postponed), open
**Outcome / hot wash** from the Event sheet or the **Event reviews** queue.

## Fundamental rule

The calendar becoming “past” does **not** mean you attended or completed the
Event. You must record outcomes explicitly.

## Mobile flow (7 steps)

1. Did we attend? (attendance + operational outcomes — kept separate)
2. What happened?
3. Top takeaways
4. Who did we meet? (encounter notes — does **not** create a contact)
5. What did we promise?
6. What needs follow-up? (confirm before creating a Follow-up)
7. Complete review

Use **Save draft** anytime. Validation errors do not wipe your fields.

## Attendance vs operational

- **ATTENDED** + **UNSUCCESSFUL** is valid
- **NOT_ATTENDED** ≠ **EVENT_DID_NOT_OCCUR** ≠ **CANCELLED** ≠ **POSTPONED**

## People met

Recording a name creates an **encounter note** only. To consider someone a
campaign contact, use **Review as contact** (enters consent-aware workflow —
never silent conversion).

## Mission debrief

If a Mission is linked, use Mission Debrief for Mission AAR. Event hot wash and
Mission Debrief stay separate; copy narrative only when you intentionally choose
to reference it.

## Queue

`/system/calendar/reviews` lists due/draft/reviewed/stale items. Opening the
queue creates **zero** reviews.
